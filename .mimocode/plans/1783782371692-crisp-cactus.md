# Módulo de Pedidos com Carrinho — Viking's Burguer

## Visão Geral

Implementar um sistema completo de pedidos no site, eliminando a dependência do iFood. O cliente adiciona itens ao carrinho, escolhe adicionais via modal, e finaliza o pedido com dados de entrega/pagamento — tudo enviado via WhatsApp.

## Fluxo do Usuário

1. Clica no botão **"+"** ao lado de cada item do cardápio
2. **Modal de adicionais** abre (se o item tiver opções; senão, vai direto ao carrinho)
3. Escolhe quantidade e adicionais (ex: +Bacon, +Queijo)
4. Item é adicionado ao **carrinho flutuante** (barra na parte inferior)
5. Pode abrir o carrinho para ver/resumir/alterar itens
6. Ao clicar **"Finalizar Pedido"**, abre modal com:
   - Nome do cliente
   - Endereço de entrega
   - Forma de pagamento (Dinheiro / Pix / Cartão)
   - Observações
7. Envia tudo formatado via **WhatsApp** como mensagem estruturada

## 2. Alterações no HTML (index.html)

### 2.1 Botões "+" nos cards

Substituir os botões `<a class="btn btn--card" data-item="...">Pedir Agora</a>` por um container flex com:
- Botão **"−"** (esconder se qty=0)
- Badge de **quantidade** (esconder se qty=0)
- Botão **"+"**

Manter o `data-item` e adicionar `data-price` no card wrapper para o JS ler.

```html
<div class="card__order" data-id="thor" data-name="THOR – Queijo Coalho e Geleia de Pimenta" data-price="34.90">
  <div class="card__qty" hidden>
    <button class="card__qty-btn card__qty-minus" aria-label="Remover">−</button>
    <span class="card__qty-num">0</span>
  </div>
  <button class="btn btn--card btn--add" aria-label="Adicionar ao carrinho">
    <svg><!-- plus icon --></svg> Pedir
  </button>
</div>
```

Quando o item já está no carrinho, o `card__qty` fica visível (display:flex) e o botão "+" incrementa. O "−" decrementa. Se chegar a 0, esconde o qty e volta ao estado original.

## 4. Lógica TypeScript (src/main.ts)

### 4.1 Estado Global

```typescript
interface CartItem {
  id: string;
  name: string;
  price: number;
  quantity: number;
  addons: { name: string; price: number }[];
  category: string;
}

const cart: CartItem[] = [];
```

### 4.2 Funções Principais

| Função | Descrição |
|--------|-----------|
| `openAddonsModal(menuItemId)` | Abre modal de adicionais para o item. Se item não tem addons, adiciona direto ao carrinho |
| `addToCart(item, addons, qty)` | Adiciona item ao array `cart`. Se já existe (mesmo id + mesmos addons), incrementa qty |
| `removeFromCart(index)` | Remove item pelo índice do array |
| `updateCartQty(index, delta)` | Incrementa/decrementa qty de um item. Se qty chega a 0, remove |
| `renderCartBar()` | Atualiza a barra flutuante (count + total) |
| `renderCartPanel()` | Renderiza a lista de itens no painel do carrinho |
| `openCheckout()` | Abre modal de checkout com resumo do pedido |
| `sendWhatsApp(name, address, payment, notes)` | Monta a mensagem formatada e abre wa.me |
| `updateCardButtons()` | Sincroniza os badges "+" nos cards do cardápio com o estado do carrinho |

### 4.4 Event Delegation

Usar event delegation no `document` para os botões "+" / "−" do cardápio e do carrinho, evitando listeners em cada elemento:

```typescript
document.addEventListener("click", (e) => {
  const target = e.target as HTMLElement;
  if (target.closest(".btn--add")) { /* abrir modal ou adicionar */ }
  if (target.closest(".card__qty-plus")) { /* incrementar */ }
  if (target.closest(".card__qty-minus")) { /* decrementar */ }
  // ... etc
});
```

### 4.5 Persistência

Usar `localStorage` para salvar o carrinho. Ao recarregar a página, o carrinho é restaurado.

```typescript
function saveCart() { localStorage.setItem("vikings-cart", JSON.stringify(cart)); }
function loadCart(): CartItem[] { return JSON.parse(localStorage.getItem("vikings-cart") || "[]"); }
```
